# MonkeyCode 传输工作台 UI 升级方案

## 1. 现状诊断

现有产品已经具备完整的文件工作流：访问验证、服务状态、上传队列、文件筛选、卡片/列表切换、批量操作、消息时间线、预览、存储统计、审计事件与深色模式。主要问题不在功能缺失，而在信息架构和视觉权重。

- 五个纵向大面板依次堆叠，上传、文件库、消息和运营状态相互距离过远，频繁操作需要持续滚动。
- 所有面板都使用大圆角、玻璃背景和重阴影，层级表达方式相同，导致主任务不突出。
- 主蓝同时承担背景氛围、按钮、标签、聚焦和状态表达，语义不够清晰。
- 中英文标题混用，工作流名称偏技术实现，缺少面向个人开发者的任务导向。
- 筛选、批量操作和时间线操作虽完整，但状态反馈分散，首次使用时不易形成操作预期。
- 移动端主要是桌面模块纵向堆叠，没有重新定义移动场景下最重要的上传、查找和最近活动。

## 2. 重构目标

1. 保留现有能力和关键 DOM 交互模型，不改变后端 API 的职责边界。
2. 把“上传与发送”设为唯一主操作，把“查找与管理”设为连续的第二任务。
3. 让连接、校验、上传和错误状态拥有独立语义，不再全部依赖主蓝。
4. 桌面端减少页面滚动，移动端重排而不是缩小。
5. 组件和状态可直接映射回现有 `composer`、`library`、`timeline`、`operations` 模块。

## 3. 新信息架构

### 桌面端

- 固定左侧栏：工作台、全部文件、活动记录、连接设备、设置。
- 顶部状态栏：当前工作区、连接状态刷新、主题切换。
- 主列：快速传输、上传队列、最近文件与筛选。
- 右侧上下文栏：连接状态、最近活动、附带消息。
- 批量操作：仅在选中文件后浮现，避免长期占据界面。
- 文件预览：右侧抽屉，不中断当前筛选和滚动位置。

### 移动端

- 顶部只保留品牌与主题切换。
- 主任务单列显示，快速传输优先于文件库。
- 底部导航承载工作台、文件、活动和设置。
- 文件表格隐藏低优先级列，并把大小、时间并入文件副标题。
- 批量操作停靠在底部导航上方，预览抽屉全宽显示。

## 4. 视觉系统

- 背景：沿用现有浅蓝灰基线，去除整页装饰渐变。
- 表面：以不透明白色和发丝边框建立层级；阴影只用于抽屉、浮动批量栏和通知。
- 主色：保留现有 `#2059eb` 的品牌识别，仅用于主按钮和当前导航。
- 辅助色：青绿色表示在线、校验通过与上传完成；红色仅用于删除和错误。
- 圆角：从 24px 普遍大圆角收敛到 10–14px，形成更精确的开发工具气质。
- 字体：Outfit 继续承担产品名和标题；系统字体负责正文；等宽字体负责文件大小、时间、延迟和容量。

## 5. 关键交互升级

- 上传：拖放和文件选择共用一个高优先级入口，上传队列直接在入口下方展开。
- 筛选：搜索保持常驻，高级筛选按需展开；清空筛选后焦点返回搜索框。
- 批量操作：选择后浮现，数量实时更新；删除前二次确认。
- 预览：使用右侧抽屉，支持遮罩点击和 Escape 关闭，并把焦点还给触发按钮。
- 消息：支持按钮发送和 `⌘/Ctrl + Enter` 快捷键，发送后立即写入最近活动。
- 状态刷新：顶部与右侧连接卡共用同一反馈，刷新过程展示“检查中”。
- 主题：遵循系统首选项并持久化用户选择。
- 可访问性：44px 最小操作尺寸、清晰焦点环、语义化表格与对话框、减少动画偏好。

## 6. 与现有代码的迁移映射

| 现有模块 | 新界面位置 | 迁移方式 |
|---|---|---|
| `composer.js` | 快速传输 + 附带消息 | 保留队列与拖放逻辑，拆分为文件入口和消息入口两个视觉区 |
| `library.js` | 最近文件 | 保留筛选、选择、分页、复制、下载、删除与事件合并 |
| `timeline.js` | 最近活动 / 独立活动页 | 工作台显示最近记录，完整时间线后续作为独立屏幕 |
| `app.js` 健康检查 | 顶部状态 + 连接卡 | 两个入口调用同一刷新函数，统一状态文案 |
| 运营摘要 | 侧栏容量 + 后续设置页 | 工作台只保留高频状态，详细审计移至独立管理屏幕 |
| 预览模态框 | 右侧预览抽屉 | 保留复制和下载能力，改善上下文连续性与焦点恢复 |

## 7. 实施顺序

1. 先替换全局令牌、按钮、输入框、表格和状态组件，不动数据逻辑。
2. 重组主页面 DOM，将 composer、library 和最近 timeline 内容放入新网格。
3. 把完整时间线和运营审计拆成独立路由，工作台只保留摘要。
4. 接回现有 API、SSE 与会话锁定逻辑，验证刷新、分页和焦点恢复。
5. 在 360、390、430、768、1024、1366、1440 和 1920px 下执行回归。
6. 最后进行键盘操作、读屏标签、错误状态和低速上传测试。

## 8. 验收标准

- 现有核心功能无缺失，上传、筛选、预览、批量操作和消息发送路径可完成。
- 1366px 桌面首屏同时看到上传入口、文件库首批内容和连接状态。
- 390px 移动端无横向滚动，主操作与底部导航触控目标不小于 44px。
- 主蓝每个区域不超过两个高显著用途；成功、在线、危险状态不再混用主蓝。
- 所有对话框和抽屉可用 Escape 关闭并恢复焦点。
- 深浅主题下正文、控件与边框保持可辨识对比。
