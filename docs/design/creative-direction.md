# MonkeyCode 创意方向与交付标准

## 什么是好看的设计

好看不是增加装饰，而是让个人开发者在三秒内确认连接是否正常、下一步该做什么、文件将去往哪里，并且能从截图中认出这是 MonkeyCode，而不是任意文件后台。

## 受众与业务目标

- 受众：频繁在本机、开发服务器和协作设备之间传输构建产物、配置与设计稿的个人开发者。
- 核心任务：快速上传和发送，其次是查找、确认、复制链接与追踪结果。
- 业务目标：降低传输不确定感，让状态透明、错误可恢复、重复操作更短。
- 成功标准：首屏可完成上传；连接、校验、目标设备和完成状态无需解释即可理解。

## 品牌气质

- 精确：边界清楚、状态明确、数字可信，不用玻璃效果掩盖层级。
- 轻快：操作反馈迅速，文案简短，动效只解释状态变化。
- 有陪伴感：猴子形象提供识别和亲和力，但保持线性、几何、开发工具化，不能卡通幼稚。
- 克制：主蓝只承担动作和当前位置；青绿只承担连接、校验和完成。

## 风格参考

- 参考现代开发者工具的高密度、发丝边框、稳定网格和等宽数据表达。
- 参考成熟文件工具对上传队列、文件表格和批量操作的连续任务设计。
- 不复刻任何单一产品的品牌元素；MonkeyCode 的识别来自猴子信号标识、传输路径和任务文案。

## 信息密度与布局

- 桌面端保持固定导航、主任务列和上下文栏，首屏同时看到上传、最近文件与连接状态。
- 移动端重排为单列任务流，文件副信息合并，底部导航保留 44px 以上触控目标。
- 上传区可以更舒展；文件表和活动列表保持紧凑。不同密度形成节奏，不使用等尺寸卡片铺满页面。
- 每个屏幕只允许一个最高权重动作。

## 色彩、字体、动效与素材

- 色彩：沿用 `brand-spec.md` 的浅蓝灰中性层、品牌蓝、连接青绿和错误红；深色主题避免纯黑纯白。
- 字体：Outfit 风格标题负责识别，系统字体负责阅读，等宽字体负责文件、容量、时间和延迟。
- 动效：按钮按压 140ms、抽屉 180ms、上传进度 280ms；遵循减少动画偏好，不使用循环装饰动画。
- 素材：优先使用 1.7px 线性 SVG 图标与真实文件数据；不依赖外部图片、插画或占位 CDN。
- 品牌资产：猴子标识必须在 24–34px 仍可辨识，轮廓简洁，耳、眼与口鼻形成最小识别结构。

## 禁止出现

- 紫蓝“可信科技”渐变、发光背景、玻璃卡片堆叠。
- 通用 `MC` 字母方块、emoji 功能图标和无意义装饰图形。
- 所有卡片同圆角、同间距、同权重的模板化节奏。
- 无来源的速度、稳定性或效率数字。
- 设计器设置、视口切换、主题参数等非产品控件。
- 只有瞬时 Toast、没有稳定加载、错误、空和禁用状态的交互。

## 资源选择

- `index.html`：唯一产品界面真相，保留已完成的响应式、交互和可访问性结构。
- `brand-spec.md`：锁定现有产品色彩和字体，不重新发明品牌。
- `redesign-plan.md`：约束信息架构与功能迁移，防止创意打磨破坏功能。
- `critique.json`：作为创意与交付质量的收敛门槛。
- 创意总监流程：负责品牌专属性、视觉记忆点和资源取舍。
- 外部生成、MCP 素材和额外插件本轮不采用：界面不缺图片资产，引入素材会增加装饰并偏离开发工具气质。

## 本轮决定性改动

1. 将通用 `MC` 方块替换为小尺寸仍清晰的线性猴子信号标识。
2. 在上传入口内加入“当前设备 → SHA-256 → 目标设备”的真实传输路径。
3. 为选择文件按钮补齐持续上传、禁用和完成后的继续添加状态。
4. 让工作台、文件与活动导航真正定位到对应任务区，并同步当前项语义。

## 验证门槛

- 320–430px 不因新增品牌或传输路径产生横向滚动。
- 脚本可解析、原生 ID 唯一、导航目标全部存在。
- 上传按钮在处理期间不可重复触发，结束后恢复可用。
- 无禁用靛紫色、填充文案、外部占位图片和装饰性渐变。
- 键盘焦点、减少动画、深浅主题与现有错误/空状态保持有效。
